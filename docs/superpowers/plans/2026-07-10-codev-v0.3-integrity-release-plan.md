# CO-DEV v0.3 Integrity Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bind every human decision to its exact gate, validate the complete CO-DEV state model, self-govern the repository, align plugin versions, and verify the checker on Windows, macOS, and Linux.

**Architecture:** `scripts/codev.ps1` becomes the single state parser and CLI for `check`, `status`, and `approve`. The existing `scripts/codev-check-gate.ps1` remains as a compatibility wrapper with no independent parsing logic. Tests execute the CLI in child PowerShell processes so exit codes are verified consistently under Windows PowerShell 5.1 and PowerShell 7.

**Tech Stack:** PowerShell 5.1-compatible syntax, PowerShell 7 cross-platform execution, GitHub Actions, JSON plugin manifests, Markdown project state.
